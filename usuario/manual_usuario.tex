\section{Introducci\'on}

MaravIA Bot es un asistente de WhatsApp que permite registrar operaciones contables (ventas y compras) de forma conversacional. El bot gu\'ia al usuario paso a paso para completar todos los datos necesarios y emitir el comprobante electr\'onico.

\subsection{Requisitos}

\begin{itemize}
    \item Un n\'umero de WhatsApp vinculado a la plataforma Maravia
    \item Acceso a la empresa configurada en el sistema
    \item Los datos de la operaci\'on (cliente/proveedor, monto, tipo de documento, etc.)
\end{itemize}

% ============================================================
% FLUJO GENERAL
% ============================================================
\section{Flujo general}

El proceso de registro tiene \textbf{4 etapas} principales:

\begin{enumerate}[label=\textbf{Etapa \arabic*:}, leftmargin=3.5cm]
    \item \textbf{Inicio} --- Indicar si es una venta o una compra
    \item \textbf{Datos del comprobante} --- Proporcionar cliente/proveedor, monto, productos, tipo de documento, moneda y m\'etodo de pago
    \item \textbf{Opciones operativas} --- Elegir sucursal, centro de costo (solo compras) y forma de pago
    \item \textbf{Emisi\'on} --- Confirmar y emitir el comprobante
\end{enumerate}

El bot acepta m\'ultiples formatos de entrada:

\begin{itemize}
    \item \textbf{Texto}: mensajes escritos con los datos de la operaci\'on
    \item \textbf{Audio}: notas de voz que el sistema transcribe autom\'aticamente
    \item \textbf{Documento (PDF)}: se almacena como documento adjunto al registro
    \item \textbf{Imagen}: se almacena como documento adjunto al registro
\end{itemize}

El bot solo pregunta por datos que faltan. Si el usuario proporciona varios datos en un solo mensaje, el bot los registra todos y solo pregunta por lo pendiente.

\section{Registro de una venta}

\subsection{Paso 1: Iniciar la conversaci\'on}

\begin{mensajeusuario}
Hola
\end{mensajeusuario}

\begin{mensajebot}
\textit{El bot saluda y muestra dos botones:}
\begin{itemize}[label=\textbullet]
    \item Registrar compra
    \item Registrar venta
\end{itemize}
\end{mensajebot}

El usuario tambi\'en puede escribir directamente: \texttt{"Quiero registrar una venta"} sin necesidad de los botones.

\subsection{Paso 2: Proporcionar los datos}

El bot pedir\'a los datos que falten. Se pueden enviar en cualquier orden y en uno o varios mensajes.

\subsubsection{Datos obligatorios}

\begin{center}
\begin{tabular}{>{\bfseries}l p{8cm}}
\toprule
Campo & Descripci\'on \\
\midrule
Cliente & Nombre o raz\'on social del cliente \\
RUC / DNI & Documento del cliente (8 o 11 d\'igitos). Opcional si es nota de venta o si el monto es menor a S/ 700 \\
Tipo de documento & Factura, boleta, recibo por honorarios o nota de venta \\
Monto o productos & Monto total o lista de productos con precios \\
Moneda & Soles (PEN) o d\'olares (USD) \\
M\'etodo de pago & Contado o cr\'edito \\
\bottomrule
\end{tabular}
\end{center}

\subsubsection{Productos y cat\'alogo}

Al mencionar un producto, el bot busca autom\'aticamente en el cat\'alogo de la empresa:

\begin{itemize}
    \item \textbf{Si encuentra un solo producto}: se registra autom\'aticamente con su precio oficial y aparece un \checkmark{} al lado del nombre en el resumen.
    \item \textbf{Si encuentra varios productos}: el bot env\'ia una \textbf{lista interactiva de WhatsApp} para que el usuario elija cu\'al es el correcto. Cada opci\'on muestra el nombre y el precio.
    \item \textbf{Si no encuentra coincidencias}: el producto se registra tal cual, sin enlace al cat\'alogo. El usuario deber\'a indicar el precio manualmente.
\end{itemize}

\begin{mensajeusuario}
Venta de 2 laptops
\end{mensajeusuario}

\begin{mensajebot}
\textit{El bot muestra una lista interactiva:}
\begin{itemize}[label=\textbullet]
    \item Laptop S/111.00
    \item Laptop Dell S/800.00
    \item Laptop Gamer X1 S/4500.00
\end{itemize}
\end{mensajebot}

El usuario selecciona una opci\'on y el bot contin\'ua el flujo mostrando el resumen con el producto elegido.

\begin{notaimportante}
Se pueden registrar \textbf{productos del cat\'alogo y productos libres en la misma operaci\'on}. Por ejemplo: 1 laptop (del cat\'alogo) y 1 servicio de instalaci\'on (libre, sin cat\'alogo). El bot maneja ambos sin problema.
\end{notaimportante}

\subsubsection{Precio del usuario vs.\ precio del cat\'alogo}

Si el usuario indica un precio expl\'icito, este se respeta aunque el cat\'alogo tenga otro precio.

\begin{mensajeusuario}
2 laptops a 1000 cada una
\end{mensajeusuario}

En este caso, el bot usar\'a S/ 1,000.00 como precio unitario, no el del cat\'alogo.

Si \textbf{no} se indica precio, el bot usa el precio del cat\'alogo autom\'aticamente.

\subsubsection{Nombres entre comillas}

Si el nombre del producto o cliente tiene varias palabras, se recomienda escribirlo entre comillas para que el bot lo trate como un solo t\'ermino:

\begin{mensajeusuario}
Venta de 1 ``Laptop Gamer X1''
\end{mensajeusuario}

Esto evita que el bot separe las palabras e intente buscar cada una por separado.

\subsubsection{Moneda y s\'imbolos}

El bot reconoce autom\'aticamente la moneda por el s\'imbolo:

\begin{center}
\begin{tabular}{>{\bfseries}l l}
\toprule
S\'imbolo & Moneda \\
\midrule
S/ & Soles (PEN) \\
\$ o US\$ & D\'olares (USD) \\
\bottomrule
\end{tabular}
\end{center}

Si el usuario escribe ``S/ 500'' o ``500 soles'', el bot asigna PEN autom\'aticamente. Si escribe ``\$200'' o ``200 d\'olares'', asigna USD. Si no se indica moneda, el bot preguntar\'a.

\subsubsection{Identificaci\'on del cliente o proveedor}

El bot necesita identificar al cliente (venta) o proveedor (compra) mediante su \textbf{RUC o DNI}. Existen tres escenarios:

\begin{enumerate}
    \item \textbf{El RUC/DNI ya est\'a registrado}: el bot devuelve autom\'aticamente el nombre asociado. No es necesario escribirlo.

    \item \textbf{El RUC/DNI no est\'a registrado}: si env\'ias el n\'umero junto con el nombre de la persona o empresa, el bot lo registrar\'a autom\'aticamente para futuras operaciones.

    \item \textbf{Error tipogr\'afico en el nombre}: si al registrar se cometi\'o un error en el nombre, la correcci\'on debe hacerse desde la \textbf{p\'agina web de Maravia} editando al cliente o proveedor directamente.
\end{enumerate}

\begin{mensajeusuario}
El cliente es Juan P\'erez, DNI 12345678
\end{mensajeusuario}

\begin{mensajebot}
\texttt{Cliente encontrado: Juan P\'erez (12345678).}
\end{mensajebot}

Si no est\'a registrado:

\begin{mensajebot}
\texttt{Cliente registrado: Juan P\'erez (12345678).}
\end{mensajebot}

\subsubsection{C\'alculo autom\'atico de IGV}

Para facturas y boletas, el bot calcula el IGV (18\%) autom\'aticamente:

\begin{itemize}
    \item Por defecto, el monto que indicar el usuario \textbf{incluye} IGV. El bot desglosa: base = monto / 1.18, IGV = monto -- base.
    \item Si el usuario dice ``m\'as IGV'', ``sin IGV'' o ``base'', el monto se trata como \textbf{base imponible} y se calcula: total = monto $\times$ 1.18.
\end{itemize}

Para notas de venta, notas de compra y recibos por honorarios, \textbf{no se calcula IGV}.

\subsubsection{Ejemplo de conversaci\'on}

\begin{mensajeusuario}
Venta de factura por 1500 soles a Empresa SAC, RUC 20123456789, contado
\end{mensajeusuario}

\begin{mensajebot}
\texttt{Anotado.}

\medskip
\texttt{VENTA} \\
\texttt{Factura} \\
\texttt{Cliente: Empresa SAC} \\
\texttt{RUC: 20123456789} \\
\texttt{Subtotal: S/ 1,271.19} \\
\texttt{IGV: S/ 228.81} \\
\texttt{Total: S/ 1,500.00} \\
\texttt{Moneda: PEN} \\
\texttt{M\'etodo: Contado}

\medskip
\texttt{Cliente encontrado: Empresa SAC (20123456789).}

\medskip
\texttt{Confirmar todo para continuar?}
\end{mensajebot}

Si faltaran datos, el bot mostrar\'ia:

\begin{mensajebot}
\texttt{Por favor, br\'indame estos datos:} \\
\texttt{1. Moneda: en soles (PEN) o d\'olares (USD)?} \\
\texttt{2. M\'etodo de pago: contado o cr\'edito?}
\end{mensajebot}

\subsection{Paso 3: Confirmar el registro}

Cuando todos los datos est\'an completos, el usuario confirma:

\begin{mensajeusuario}
Confirmo
\end{mensajeusuario}

Palabras aceptadas: \textit{s\'i, confirmo, dale, correcto, listo, ok, de acuerdo, va, perfecto, adelante, acepto, vale, est\'a bien, procede}.

\subsection{Paso 4: Elegir opciones operativas}

Tras confirmar, el bot presenta listas interactivas de WhatsApp para elegir:

\begin{enumerate}
    \item \textbf{Sucursal} --- Lista de sucursales de la empresa
    \item \textbf{Forma de pago} --- Transferencia, efectivo, tarjeta, billetera virtual, etc.
\end{enumerate}

\begin{mensajebot}
\textit{El bot env\'ia una lista interactiva con las sucursales disponibles.}
\end{mensajebot}

El usuario selecciona una opci\'on de la lista y el bot avanza al siguiente campo.

Despu\'es de elegir todas las opciones:

\begin{mensajebot}
\texttt{Por favor, br\'indame una confirmaci\'on para continuar}
\end{mensajebot}

\subsection{Paso 5: Confirmaci\'on final y emisi\'on}

\begin{mensajeusuario}
Dale
\end{mensajeusuario}

\begin{mensajebot}
\texttt{VENTA REGISTRADA EN SUNAT!}

\medskip
\texttt{Cliente: Empresa SAC} \\
\texttt{Total: S/ 1,500.00} \\
\texttt{Documento: F001-00001234}

\medskip
\texttt{Te enviamos el comprobante en el siguiente mensaje.}

\medskip
\textit{[Se env\'ia el PDF del comprobante]}
\end{mensajebot}

\section{Recibo por honorarios}

El recibo por honorarios se usa para registrar servicios profesionales. Tiene reglas especiales:

\begin{itemize}
    \item \textbf{Sin IGV}: el monto es el bruto del honorario, sin desglose de impuestos
    \item \textbf{Requiere RUC}: se necesita el RUC (11 d\'igitos) del profesional que presta el servicio
    \item \textbf{Serie/n\'umero}: se puede indicar el n\'umero del recibo (formato E001-XXXX)
\end{itemize}

\begin{mensajeusuario}
Compra de recibo por honorarios por 2000 soles a Nicol\'as Castro, RUC 10752397908, contado
\end{mensajeusuario}

\begin{mensajebot}
\texttt{COMPRA} \\
\texttt{Recibo por honorarios} \\
\texttt{Proveedor: Nicol\'as Castro} \\
\texttt{RUC: 10752397908} \\
\texttt{Total: S/ 2,000.00} \\
\texttt{Moneda: PEN} \\
\texttt{M\'etodo: Contado}

\medskip
\texttt{Por favor, br\'indame estos datos:} \\
\texttt{1. N\'umero del recibo (serie-n\'umero, ej: E001-18)?}
\end{mensajebot}

Para activar este tipo de comprobante, se puede decir: ``honorarios'', ``recibo por honorarios'' o ``recibo de honorarios''. Si solo se dice ``recibo'' sin m\'as contexto, el bot lo interpreta como boleta.

\section{Notas de venta y notas de compra}

Las notas son documentos internos con reglas especiales:

\begin{itemize}
    \item \textbf{No se calcula IGV}: el monto total es el dato principal, sin desglose de impuestos
    \item \textbf{No se pide n\'umero de serie}: el sistema no pregunta por serie-n\'umero del comprobante
    \item \textbf{Documento del cliente/proveedor es opcional}: no se exige RUC ni DNI
\end{itemize}

\begin{mensajeusuario}
Nota de venta por 200 soles
\end{mensajeusuario}

\begin{mensajebot}
\texttt{VENTA} \\
\texttt{Nota de venta} \\
\texttt{Total: S/ 200.00}

\medskip
\texttt{Por favor, br\'indame estos datos:} \\
\texttt{1. Cliente: nombre o raz\'on social?} \\
\texttt{2. M\'etodo de pago: contado o cr\'edito?}
\end{mensajebot}

\subsection{Cuando no se tiene el documento del cliente}

Si durante cualquier registro el usuario indica que no tiene el RUC o DNI del cliente (por ejemplo: ``no tengo su RUC'', ``no s\'e el DNI''), el bot autom\'aticamente:

\begin{enumerate}
    \item Cambia el tipo de documento a \textbf{nota de venta} o \textbf{nota de compra}
    \item Guarda el nombre del cliente como referencia en las observaciones
    \item Contin\'ua el flujo sin pedir documento de identidad
\end{enumerate}

% ============================================================
% REGLA DE LOS S/ 700
% ============================================================
\section{Regla de los S/ 700}

Para facturas y boletas en soles:

\begin{itemize}
    \item Si el monto total es \textbf{menor a S/ 700}: el RUC o DNI del cliente es \textbf{opcional}
    \item Si el monto total es \textbf{igual o mayor a S/ 700}: el RUC o DNI es \textbf{obligatorio}
\end{itemize}

Esta regla \textbf{no aplica} a notas de venta ni notas de compra (el documento siempre es opcional en notas).

\section{M\'etodo de pago: contado vs cr\'edito}

\subsection{Contado}

Si el usuario indica \texttt{contado}, el bot lo registra y \textbf{no hace preguntas adicionales} sobre d\'ias de cr\'edito ni cuotas. Sin\'onimos aceptados: ``efectivo'', ``cash'', ``al toque'', ``de una'', ``pago directo''.

\subsection{Cr\'edito}

Si el usuario indica \texttt{cr\'edito}, el bot preguntar\'a:

\begin{mensajebot}
\texttt{1. A cu\'antos d\'ias ser\'a el plazo? (15, 30, 45, 60 o 90 d\'ias)} \\
\texttt{2. En cu\'antas cuotas ser\'a el pago? (de 1 a 24)}
\end{mensajebot}

Sin\'onimos aceptados: ``a plazos'', ``financiado'', ``a cuotas'', ``me f\'ian''.

\begin{notaimportante}
Una vez registrado el m\'etodo de pago, el bot \textbf{no volver\'a a preguntar} por este campo. Si necesita cambiarlo, simplemente escriba el nuevo valor (por ejemplo: \texttt{``cambiar a cr\'edito''}).
\end{notaimportante}

% ============================================================
% DOCUMENTO ADJUNTO

\section{Documento adjunto}

Cuando el usuario env\'ia un \textbf{documento PDF} o una \textbf{imagen} por WhatsApp, el bot lo almacena autom\'aticamente como documento adjunto de la operaci\'on:

\begin{itemize}
    \item El archivo se guarda y \textbf{se preserva} durante todo el flujo, sin importar cu\'antos mensajes se env\'ien despu\'es
    \item Se vincula autom\'aticamente al comprobante al finalizar la operaci\'on
    \item Solo se elimina cuando se borra o cancela el registro completo
    \item Si se env\'ia un nuevo documento, reemplaza al anterior
\end{itemize}

\begin{notaimportante}
No es necesario enviar el documento en un momento espec\'ifico del flujo. Se puede enviar al inicio, durante la carga de datos o justo antes de confirmar.
\end{notaimportante}

Si el usuario tiene dudas sobre qu\'e datos proporcionar o en qu\'e paso est\'a, puede pedirle al bot que le explique. El bot responde con informaci\'on clara sobre el estado actual del registro y los siguientes pasos.

\subsection{Cancelar operaci\'on}

Al cancelar, el registro se elimina por completo y el bot confirma:

\begin{mensajebot}
\texttt{Entendido. He cancelado la operaci\'on y limpiado el borrador. Deseas iniciar un registro nuevo?}
\end{mensajebot}

\section{Errores y reintentos}

\subsection{Si el bot no responde}

Si env\'ias un mensaje y \textbf{no recibes ninguna respuesta} (ni de \'exito ni de error), esto indica un \textbf{problema de conexi\'on}, no un fallo del bot. Simplemente \textbf{vuelve a enviar tu mensaje} y el bot continuar\'a el flujo normalmente desde donde qued\'o.

\subsection{Errores al emitir}

Si ocurre un error al emitir el comprobante (por ejemplo, un error de SUNAT, stock insuficiente o datos inv\'alidos), el bot:

\begin{enumerate}
    \item Muestra el mensaje de error espec\'ifico
    \item Regresa al paso de edici\'on (antes de la confirmaci\'on)
    \item Permite corregir los datos problem\'aticos
    \item El usuario puede volver a confirmar e intentar nuevamente
\end{enumerate}

\begin{mensajebot}
\texttt{Error SUNAT: La fecha de emisi\'on no puede ser anterior a 3 d\'ias.}

\medskip
\texttt{Reinici\'e el flujo al paso de edici\'on. Puedes actualizar los datos y volver a confirmar para intentar nuevamente.}
\end{mensajebot}

\begin{mensajeusuario}
La fecha de emisi\'on es hoy
\end{mensajeusuario}

\subsection{Error de stock insuficiente}

Si se intenta vender un producto del cat\'alogo y no hay stock suficiente en la sucursal seleccionada, el bot mostrar\'a un mensaje indicando cu\'antas unidades faltan. El usuario puede cambiar la cantidad, la sucursal o el producto.

\section{Preguntas frecuentes}

\begin{description}[style=nextline]
    \item[Puedo enviar todos los datos en un solo mensaje?]
    S\'i. El bot extrae autom\'aticamente todos los datos que encuentre en el mensaje y solo pregunta por lo que falta.

    \item[Puedo cambiar un dato despu\'es de haberlo ingresado?]
    S\'i. Simplemente escribe el nuevo valor (por ejemplo: ``el monto es 2000'' o ``cambiar moneda a d\'olares'').

    \item[Puedo registrar varios productos en una sola operaci\'on?]
    S\'i. Se pueden agregar productos del cat\'alogo y productos libres en la misma operaci\'on. Al agregar un nuevo producto, no se borran los anteriores.

    \item[Qu\'e pasa si el bot no responde?]
    Si no recibes un mensaje de error, probablemente se trat\'o de un \textbf{problema de conexi\'on}. Simplemente vuelve a enviar tu mensaje y el bot continuar\'a desde donde qued\'o.

    \item[Qu\'e pasa si el RUC/DNI no est\'a registrado?]
    Si env\'ias un RUC o DNI junto con el nombre de la persona o empresa, el bot lo \textbf{registrar\'a autom\'aticamente} como cliente (en ventas) o proveedor (en compras) para futuras operaciones.

    \item[Qu\'e pasa si hay un error en el nombre del cliente o proveedor?]
    Si hubo un error tipogr\'afico al registrar el nombre, este debe corregirse desde la \textbf{p\'agina web de Maravia} editando directamente al cliente o proveedor.

    \item[D\'onde queda registrada la operaci\'on?]
    Las ventas quedan en la \textbf{tabla de ventas} y las compras en la \textbf{tabla de compras} del sistema Maravia, disponibles para consulta desde la plataforma web.

    \item[Puedo registrar varias operaciones a la vez?]
    No. Cada conversaci\'on maneja una operaci\'on a la vez. Debes finalizar o cancelar la actual antes de iniciar otra.

    \item[C\'omo cancelo una operaci\'on?]
    Escribe ``cancelar'', ``borrar'' o ``empezar de cero''. El registro se eliminar\'a por completo.

    \item[Qu\'e diferencia hay entre ``recibo'' y ``recibo por honorarios''?]
    Si solo dices ``recibo'', el bot lo interpreta como \textbf{boleta}. Para registrar un recibo por honorarios, debes decir expl\'icitamente ``recibo por honorarios'' u ``honorarios''.
\end{description}

\section{D\'onde queda registrada la operaci\'on}

Al finalizar exitosamente:

\begin{itemize}
    \item Las \textbf{ventas} quedan registradas en la \textbf{tabla de ventas} del sistema Maravia, con su comprobante electr\'onico emitido ante SUNAT (cuando corresponda). El bot env\'ia el PDF del comprobante por WhatsApp.
    \item Las \textbf{compras} quedan registradas en la \textbf{tabla de compras} del sistema Maravia. El bot env\'ia un mensaje de confirmaci\'on por WhatsApp (sin PDF).
\end{itemize}

Ambas son consultables desde la \textbf{plataforma web de Maravia}. El registro temporal en el bot se elimina autom\'aticamente una vez que la operaci\'on se completa con \'exito.
